| Требование (из методички) | Что должно быть в записке | Чем подтверждается в проекте (файлы/окна/классы) | Статус | Что нужно добавить |
|---|---|---|---|---|
| A) Введение | Цель работы, актуальность, краткое описание системы и результатов | `main.cpp` (GUI entrypoint), `loginwindow.*`, наличие ролей (см. `Database::findUser`) | PARTIAL | Добавить формулировку цели/задач (3–6 пунктов) и ожидаемый эффект |
| A) Анализ предметной области | Описание процессов (учёт оценок/пропусков/расписания), акторы | Роли: admin/teacher/student (таблица `users.role`), окна `AdminWindow`, `TeacherWindow`, `StudentWindow` | PARTIAL | Описать бизнес-процессы (as-is/to-be), ограничения, терминологию |

| A) Постановка задачи | Требования (функциональные/нефункциональные), границы проекта | Частично по UI и методам `Database` | PARTIAL | Выписать требования: роли, операции, хранение данных, безопасность (логин/пароль), производительность/офлайн |

| B) Диаграмма классов / компонентов | UML (классы основных окон и DB), связи | Заголовки: `loginwindow.h`, `adminwindow.h`, `teacherwindow.h`, `studentwindow.h`, `database.h` | PARTIAL | Нарисовать UML (можно упрощённо) для окон и `Database` |
| B) Модель данных / структура хранения | Таблицы, ключи, связи, индексы | `Database::initialize()` в `database.cpp` (CREATE TABLE), `database.h` (API) | OK | Оформить в записке: ER-диаграмма + таблицы/поля/связи |
| B) Реализация (фрагменты кода) | Обоснование выбора технологий, ключевые алгоритмы, фрагменты | Примеры: авторизация `Database::findUser`, расписание `getScheduleForGroup`, журнал `upsertGradeByKey`, статистика `Statistics::calculateStudentAverage` | PARTIAL | Вставить 3–6 фрагментов кода с объяснениями (почему так) |

| B) Тестирование / проверка | Сценарии, результаты, скриншоты | Приложение имеет окна; в репо есть `tests/*` (но не обязательно для сдачи) | PARTIAL | Добавить 5–10 тест-сценариев (ручных) + скриншоты окон |

| C) Заключение | Итоги, достигнутые цели, направления развития | (в проекте нет) | MISSING | Добавить выводы + план улучшений (экспорт отчётов, роли/права, UX) |
