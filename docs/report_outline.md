# Глава 2. Проектирование и реализация (B)
2.1. Общая архитектура
- схема модулей/компонентов
- взаимодействие: UI (Qt) -> Data access (`Database`) -> SQLite

2.2. Проектирование пользовательского интерфейса
- перечень окон
- навигация по ролям
- принципы UX/ограничения

2.3. Проектирование модели данных
- описание таблиц
- ключи и связи
- индексы

**Место для обязательного “чертежа/схемы” (A4, по СТП):**
- вариант 1: ER-диаграмма БД (рекомендуется)
- вариант 2: схема модулей/компонентов (UI/DB/SQLite)

2.4. Проектирование классов и компонентов
- диаграмма классов (окна + `Database`)
- ключевые структуры данных

2.5. Реализация (фрагменты кода)
- авторизация (поиск пользователя)
- работа с расписанием
- журнал оценок/пропусков (insert/update)
- расчёт статистики

2.6. Обоснование выбора технологий
- Qt Widgets
- SQLite (embedded)
- CMake

2.7. Сборка и развертывание
- требования к окружению
- как собрать проект
